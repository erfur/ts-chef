# Design: Recipe default font size

**Date:** 2026-07-17
**Status:** Approved

## Problem

The Recipe webview hard-codes an `11px` font size for argument labels and
argument text, number, and select controls. Other recipe fields use browser
defaults. This makes field typography inconsistent and prevents the full view
from following VS Code's configured default font size.

## Design

Keep the change within the embedded stylesheet generated by
`RecipeViewProvider`:

- Set the `body` font size to `var(--vscode-font-size)` alongside its existing
  VS Code font family.
- Make `input`, `select`, and `button` elements use `font: inherit` so native
  form-control defaults cannot override the webview typography.
- Remove the two hard-coded `font-size: 11px` declarations from argument labels
  and controls.

This applies the configured VS Code default font family and size to the recipe
name, step text, argument labels, argument inputs and selects, and action
buttons. Existing colors, spacing, layout, interactions, and webview message
flow remain unchanged.

## Testing

Extend `test/commands/recipeViewProvider.test.ts` with a focused HTML/CSS
regression test that verifies the generated webview stylesheet:

- uses `var(--vscode-font-size)` for the default size;
- makes form controls inherit typography; and
- contains no hard-coded `font-size: 11px` declaration.

No runtime error handling or data-flow changes are needed because this is a
stylesheet-only behavior change.
